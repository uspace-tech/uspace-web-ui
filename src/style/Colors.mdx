## Color Token Architecture

Our color system uses a two-tier approach:

### Raw Color Tokens
Base color values prefixed with `_`:
- `_text._primary`, `_text._secondary`, `_text._secondaryDark`
- `_materialBg`, `_materialBgDark`
- `_disabled._DEFAULT`, `_disabled._grey`
- `_primary`, `_blue`, `_green`, `_yellow`, `_red`, `_white`

### Semantic Tokens
Context-aware tokens that automatically switch between light and dark modes:
- `text.primary`: `#333333` (light) → `#FFFFFF` (dark)
- `text.secondary`: `#748598` (light) → `#CCCCCC` (dark)
- `materialBg`: `#F3F4F9` (light) → `#484848` (dark)

## Usage

### In Chakra UI Components

```tsx
import { Box, Text } from "@chakra-ui/react"

// Using semantic color tokens (recommended)
<Box bg="materialBg" borderColor="divider">
  <Text color="text.primary">Main content</Text>
  <Text color="text.secondary">Supporting information</Text>
</Box>

// Using UI colors
<Box bg="blue" color="white">
  Blue background with white text
</Box>

// Using state colors
<Button disabled bg="disabled.grey">
  Disabled Button
</Button>
```

### Dark Mode Support

All semantic tokens automatically adapt to the current color mode:

```tsx
// These will automatically switch colors in dark mode
<Text color="text.primary">Always readable main text</Text>
<Text color="text.secondary">Always readable secondary text</Text>
<Box bg="materialBg">Adaptive background</Box>
```

### Color Mode Testing

Use the color mode toggle in Storybook to test how components adapt between light and dark themes.

### Color Accessibility

- **Text Colors**: Semantic tokens ensure sufficient contrast ratios in both light and dark modes
- **Primary Text**: Always provides excellent readability against backgrounds
- **Secondary Text**: Maintains proper contrast while being visually secondary
- **UI Colors**: Use sparingly for interactive elements and status indicators

## Migration Notes

If you were using the old color tokens, update your code:
- `text` → `text.primary`
- `text.primary` → `text.primary` (no change needed)
- `text.secondary` → `text.secondary` (no change needed)
- `disabled` → `disabled.grey`
